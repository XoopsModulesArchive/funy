Attention: Cette option permet de définir de nouvelles balises dans un thème. Assurer vous d'avoir pris toutes les précautions nécessaires (sauvegarde, ..).
Pour créer une balise il faut:

  • définir un repère. C’est un morceau de chaine dans le fichier "theme.html" qui permet de situer le remplacement ou l’insertion. Puis définir le texte à insérer à la place, avant ou après. (Rappel : syntaxe d'une balise smarty  :  <{$maBalise}> ).
    La création de nouvelles balises suppose qu'un plugin utilise ces dernières.
    Pour la création d'un nouveau plugin il est préférable d'utiliser les balises déjà existantes, sauf à avoir des particularités tel qu'insérer par exemple un "div"  à un endroit précis.
  • Activer les balise que vous insérer en cochant la case sur la ligne de la balise et cliquez sur "mettre  jour"
  • Mettez le répertoire du thème que vous voulez modifier en chmod "777"
  • Mettez le fichier  theme.html" th thème que vous voulez modifier en chmod "666"
  • Sélectionner le thème dans la liste déroulante en bas de la page et cliquez sur "Affecter les balises".
    Cette action crée un fichier "theme.html.old" de sauvegarde s'il n'existe pas déjà,   qui sera utilisé pour l'insertion des balises, surtout ne le supprimez pas. Ce fichier permet aussi de restaurer le fichier original en cliquant sur 'Effacer les balises'.
    Seules les balises actives seront insérées dans le fichier.
  • Remettez le fichier  "theme.html" du thème que vous avez modifié en chmod "644"
  • Remettez le répertoire du thème que vous avez modifié en chmod "755"

Attention la moindre erreur dans la syntaxe des balises peut rendre indisponible votre site; Il faudra restaurer le fichier original ou changer de thème. Ne modifiez pas les balises sans être sur de vous.
Une fois avoir affecté les balises aux thème, il est possible toutefois de revenir au fichier original en cliquant sur "effacer les balises".

Information :

  • Les balises en gras dans la liste ci-dessous sont celles actuellement présentes dans le thème courant.
  • Certaines de ces balises sont déjà présente dans le bloc "funy_action" si elles ont été insérées dans le fichier "theme.html" elle ne seront pac présente dans le bloc.
  • L'utilisation de ces balise permet notamment d'ordonnancer les bloc de code javascript dans la page (par exemple, pour mettre les déclarations avant le code des fonctions)
  • L'insertion de toutes les balises permet de ne pas afficher le bloc "funy_action" mais dans ce cas les bouton "start" et "stop" ne pourrons plus être exploités. Par contre vous pouvez toujours via un javascript, invoquer les fonctions d'arrêt ou de démarrage  propre à chaque plugin (voir le fichier de config du plugin), ou encore les fonction global du module qui appelle ces fonction:  "funy_start_all ();" équivalent au bouton "start" et "funy_stop_all ();" équivalent au bouton "stop".
