Module FUNY pour XOOPS - PRESENTATION

Funy  est un module de gestion de script côté client ( javascript) .

Ces script petmettent de gérer des animations style image qui tombe, ou qi traverse l'écran, neige ou autre

Un gestionnaire d'événement permet de gérer automatiquement, le déclenchement des animations, combien temps et le renouvellement périodique de ces dernières;
Par exemple il est possible de faire tomber de la neigne à noêl ou des cloches à pâques, en programmant les dates de début et de fin des animations.
Pour l'instant la bibliothèque d'animation n'et pas très riche, mais comme elle concue sous forme de plugin, il est tout à fait possible de développer ses propres animations et de les intégrer dans la bibliothèque.

Installation

Décompressez le fichier sur votre disque dur et copiez le contenu du répertoire "funy" à la racine de votre site.

Mettre le dossier "modules/_plugin_declaration" en chmod "777"

Donner aux groupes les autorisation d'accès aux blocs du module

Afficher le bloc "funy_action" à gauche ou a droite de préférence, et sur les pages souhaitées. Les bouton "Stop" et "Start" de ce bloc pemettent d'arrêter les animations quand les fonctions idoines ont été prévues.

Afficher le bloc "funy_text" si vous souhaitez utiliser les plugins qui l'utilise (teste_ondulant,...)

Ensuite dans l'administration de votre site, procédez comme pour les autres modules.

Options du module

  

option Description
Éditeur Les éditeurs KOIVI, TinyEditor, Inbetween, advanced DHTML et bien sûr DHTML sont reconnus. Toute fois ils doivent être installés pour être utilisés, et ne sont pas fourni avec le module.
URL de la documentation La documentation nest pas installé avec le module. Par défaut elle pointe sur le site de l'auteur, ce qui permet une mise à jour autioatique.

Toutefois 
   
   

Plugins

La liste des plugins apparait automatiquement dans l'onglet plugin de l'administration du module.

Evennements

Pour créer un évennement dans l'onglet "plugins" de l'administration du module cliquer sur l'iône  "nouveau"  droite de la liste.
renseignez le nom de l'évènement., les dates de début et de fin, ainsi que la périodicité. L'année et le mois ne sont qu'informatifs selon la périodicitée choisie; Par exempl une programmation du 20/12/2008 au 31/12/2008 avec ne périodicité annuelle sera déclenché tous les ans sans tenir compte de l'année d'orrigine.

Puis selon selon le plugin adaptez les paramètres en fonction de vos besoins.
Après validation, l'évennement apparaitra dans l'onglet "Evennements".
Vous pouvez désactivez ou réactiver un plugin en cochant la case dans la liste des évennements. attention m^me cocher, l'activation ne sera effective qu'en fonction des dates programmées.

IMPORTANT
Un seul évennement par plugin sera actif en même temps, celui dont la date de début est la plus récente. En effet les script ne sont  pas prévu pour faire tourner plusieurs instances d'un m^me plugin. Par ailleurs si les animations sont interresantes, il ne faut pas non plus en abuser.

Création d'un plugin

Fichier de configuration

Le fichier de configuration est un fichier ini avec des section et des clés.
exemple:
[info]
nom = "Laisse tomber la neige"
auteur = "Altan"
site   = "http://www.altan.hr/snow"
adaptation = "JJD"
stopScript=funy_snow_stop
runScript=funy_snow_start

[balise]
funy_body = "body.js"

;-------------------------------------------------------------------
;paramètres du script
[funy_no]
name = "funy_snow_no"
value =15
type = 1
min = 5
max = 50
 
[funy_speed]
name = "funy_snow_speed"
value =5
type = 1
min = 5
max = 1000

[funy_snowflake]
name = "funy_snow_flake"
value = "snow.gif"
type = 3
folder = "modules/funy/_ressources/"

Descriptions:
Section Clé Description
info nom Nom du plugin qui apparait dans 'onglet "plugins"
info auteur Auteur. Merci de faire référence  ce dernier même quand le script est adapter pour ce module.
info site Site de l'auteur du script
info adaptation Nom de la personne ayant adapté le script pour  le module "Funy"
info stopScript Nom de la fonction javascript à appeler pour stoper le script. Utilisé notamment dans le bloc, quand on clique sur le bouton "stop".
info runScript Nom de la fonction javascript à appeler pour lancer le script. Utilisé notamment dans le bloc, quand on clique sur le bouton "start".
balise
 Inutilisé pour le moment
Cette section contient la liste des fichiers javascript (*.js) et les blise smarty correspondante.. Chacun de ces fichier sera insére dans la balise. Ces balises sont définis dans le bloc "funy_block_action.html".


Les autres sections définissent les paramètres du plugins. Ils apparaitrond en édition de l'événnements, et permettrons de générés un javascript qui sera insérer avant le "body.js". Ces paramètres sont à utiliser notamment pour définir la vitesse de l'aimation, les texte, l'image etc.
Clé Description
name
 Nom du paramètre. c'est le nom de la variable utilisées dans les javaScript.
value Valeur par défault du paramètre
type 0 = Text
1 =  spinButton (renseigné les clé min et max pour ce type)
2 = Liste de libele dont le numéro d'ordre est atomatique (le premier commence à 0)
3 =  c'est une liste de fichiers GIF
4 = Permet de définir une couleur
5 = Texte HTML
6 = Variable masquée déclaére dans le fichier ini mais non modifiable
7 = liste de valeur dans une table défini par les clés "table" et "field" associées
8 = Url du module (http://votreSite/modules/funy/)
9 = nom de l'événnement
10 = Defini un titre. Permet de classer les paramètre dans l'interface de paramètrage des évennements.
20 = Ligne de separation. Permet de classer les paramètre dans l'interface de paramètrage des évennements.
)min Valeur minimum si type=1
max Valeur maximum si type=1
folder Répertoire des ressources , images, ou autres. (Utilisé par type=3)

Attention les éléments de configuration sont suceptibles d'évoluer en fonction des besoin des plugins.

Les fichiers de langue:

Utiliser pour les traduction des descriptions des paramêtres.
Chaque clé du fichier de langue doit correstpondre au nom des paramètres tel que défini dans le fichier "ini" (nom des clés);
exemple:
[language]
description    = "fait tomber des flocons de neiges"
funy_no        = "nombre d'images"
funy_speed     = "plus le nombre est petit, plus la vitesse augmente"
funy_snowflake = "Image a afficher"

Les fichiers JavaScript

Il doit y avoir au moins un fichier javascript, de préférence nommé "body.js".
Il sera insérer juste après le déclaration des varaibles générés par l'évennement du plugin.


Important

Afin de na pas risquer de dupliquer des nom de fonctions ou de variables, il est fortement recommandé de préfixer tous les objets d'un plugin (nom de variables, fonctions, objet DIV, etc)
Voir le plugin "neige qui tombe" en exemple.
